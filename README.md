## О проекте

`OMSK: The Gathering` это SPA-игра на React + TypeScript, где:

- есть синглплеер против ИИ ("Хранитель Омска") и локальный PvP;
- реализован игровой движок с MTG-подобными фазами и ключевыми словами;
- присутствуют синематики, эффекты карт, история мира и коллекция карт;
- игра готова к сборке и выкладке на GitHub Pages.

Смысл и полная проектная спецификация сохранены в [RECREATE_PROMPT.md](./RECREATE_PROMPT.md).

## Ключевые особенности

- 60+ уникальных карт с лором и механиками.
- ИИ-противник с приоритетами розыгрыша и атак.
- Ключевые слова: `haste`, `defender`, `flying`, `trample`, `lifelink`, `deathtouch`, `first_strike`, `hexproof`, `unblockable`.
- Кинематографическое интро, анимации розыгрыша и гибели, атмосферный UI.
- Локальный кэш обложек карт и рубашки с fallback-механизмом.

## Геймплей

### Типы карт

| Тип | Описание |
| --- | --- |
| `creature` | Существа с атакой/здоровьем, участвуют в бою |
| `spell` | Одноразовые эффекты, после розыгрыша уходят в кладбище |
| `enchantment` | Постоянные эффекты на поле |
| `land` | Источник маны, обычно 1 земля за ход |

### Фазы хода

1. Начало хода: добор, обновление ресурсов.
2. Основная фаза: розыгрыш земель/карт.
3. Бой: выбор атакующих и целей.
4. Конец хода.

### Управление (основное)

- Клик по карте: просмотр/выбор.
- Двойной клик по карте руки: быстрый розыгрыш.
- Drag and drop из руки на поле: розыгрыш карт.
- Кнопка `Конец хода`: передача инициативы.

## Быстрый старт

- Node.js `20.19+` (рекомендуется, Vite 7 требует минимум 20.19)
- npm

### Установка и запуск

```bash
npm ci

```

Открыть в браузере `http://localhost:5173`.

## Скрипты

```bash
npm run dev              # локальный dev-сервер
npm run build            # production-сборка в dist/
npm run preview          # локальный просмотр production-сборки
npm run test:regression  # регрессионные тесты движка боя
npm run cache:card-images # кэширование обложек/рубашки в public/cards
```

## Локальные обложки карт

В проекте реализован fallback-цепочка источников:

`local file -> external URL -> emoji/icon fallback`.

### Как обновить кэш обложек

1. Создать `.env` на основе [.env.example](./.env.example):

```env
POLLINATIONS_API_KEY=your_token_here
POLLINATIONS_IMAGE_MODEL=flux
```

2. Запустить:

```bash
npm run cache:card-images
```

После выполнения:

- файлы появятся в `public/cards/`;
- карта путей обновится в `src/data/localCardImages.ts`.

Важно: `.env*` исключены из git, чтобы ключи не попадали в репозиторий.

## Тестирование и качество

- Боевые регрессии: `npm run test:regression` (freeze/lifelink/deathtouch/AI и т.д.).
- Smoke-визуальные проверки через Playwright (скриншоты и отчеты в `output/...`).
- Production-сборка: `npm run build`.
