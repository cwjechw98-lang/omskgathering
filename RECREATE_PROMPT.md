## МЕХАНИКИ (полная реализация MTG)

### Типы карт:
1. **Существа** (creature) — имеют атаку/здоровье, могут атаковать/блокировать
2. **Заклинания** (spell) — одноразовый эффект, уходят на кладбище
3. **Наложения** (enchantment) — постоянный эффект на поле
4. **Земли** (land) — дают ману, 1 в ход бесплатно

### Keywords (ВСЕ должны работать в engine.ts):
- **Haste** (Ускорение) — атакует в ход призыва
- **Defender** (Защитник) — не атакует, но ОБЯЗЫВАЕТ врага атаковать себя (наземные не могут атаковать героя если есть defender)
- **Flying** (Полёт) — блокируется только летающими, обходит наземных защитников
- **Trample** (Растоптать) — избыточный урон проходит в героя
- **Lifelink** (Привязка к жизни) — урон существом = хил владельцу
- **Deathtouch** (Смертельное касание) — любой урон = мгновенная смерть цели
- **First Strike** (Первый удар) — бьёт первым, если убил — враг не отвечает
- **Vigilance** (Бдительность) — не тапается при атаке
- **Hexproof** (Порчеустойчивость) — нельзя таргетить заклинаниями
- **Unblockable** (Неблокируемый) — нельзя заблокировать

### Правила боя:
- Summoning sickness — существо не атакует в ход призыва (кроме Haste)
- Существо тапается при атаке (кроме Vigilance)
- Атака в героя блокируется защитниками (кроме Flying/Unblockable)
- При атаке существа — оба наносят урон друг другу одновременно (кроме First Strike)
- Земли: 1 бесплатная в ход, дают +1 maxMana и refill

### Фазы хода:
1. Начало хода — draw карту, untap всех, refresh маны
2. Основная фаза — играть земли, существ, заклинания
3. Боевая фаза — выбрать атакующих, выбрать цели
4. Конец хода — передать ход

## ИИ-ПРОТИВНИК ("Хранитель Омска")

### Стратегия:
1. Всегда играет землю если есть
2. Приоритет: removal (убить угрозы) → существа (по кривой маны, дорогие первые) → наложения
3. Атака: расчёт выгодных разменов, летуны бьют героя, all-in при летале
4. Защита: не атакует защитниками, сохраняет блокеров

### Комментарии ИИ:
- Каждая карта имеет 3 набора реплик по HP (>60%, 30-60%, <30%)
- При высоком HP — дерзкие ("Омск поглотит тебя!")
- При среднем — нейтральные ("Интересный ход...")  
- При низком — паникующие ("Нет... метро спаси меня...")
- Реплики показываются при розыгрыше карты ИИ

## ПРЕДОТВРАЩЕНИЕ УТЕЧЕК ПАМЯТИ

КАЖДЫЙ useEffect с таймером/RAF/интервалом ОБЯЗАН иметь cleanup:
```typescript
useEffect(() => {
  const id = requestAnimationFrame(loop);
  return () => cancelAnimationFrame(id);
}, []);

useEffect(() => {
  const timer = setTimeout(...);
  return () => clearTimeout(timer);
}, []);

useEffect(() => {
  const interval = setInterval(...);
  return () => clearInterval(interval);
}, []);
```

Для AI хода — использовать mounted ref:
```typescript
const mounted = useRef(true);
useEffect(() => { return () => { mounted.current = false; }; }, []);
// В async функциях: if (!mounted.current) return;
```

## 🔧 7 Принципов структурного программирования (применённые в коде)

### 1. Модульность
- Код разделён на модули: `engine.ts` (логика), `ai.ts` (ИИ), `types.ts` (типы), `cards.ts` (данные), `lore.ts` (нарратив)
- Каждый модуль решает одну задачу, экспортирует чёткий API

### 2. Иерархическая структура
- `App.tsx` → `MainMenu.tsx` / `StoryIntro.tsx` / `GameBoard.tsx`
- `GameBoard.tsx` → `PlayerArea.tsx`, `CardPlayAnimation`, `CardDeathAnimation`

### 3. Единственность входа/выхода
- Все функции движка (`playCard`, `attackPlayer`, `attackCreature`, `endTurn`) имеют один вход и один выход
- Возвращают либо новое состояние, либо исходное (immutable pattern)

### 4. Ограниченное использование goto
- Нет goto/label. Используются только структурные конструкции: if/else, switch/case, for/while, early return

### 5. Использование только 3 базовых конструкций
- Последовательность: линейное выполнение
- Ветвление: if/else, switch/case для карт
- Цикл: for для обработки полей, while для AI-раундов (с safety counter)

### 6. Ограниченный размер модулей
- Самый большой файл `GameBoard.tsx` ~700 строк — в пределах нормы для React-компонента
- Функции engine.ts в среднем 20-50 строк

### 7. Минимизация глобальных переменных
- Единственная глобальная переменная: `uidCounter` для генерации уникальных ID
- Всё остальное состояние передаётся через параметры или React state
