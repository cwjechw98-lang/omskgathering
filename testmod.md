Добавить property-based тестирование для игрового движка.

Цель: автоматически находить ошибки в механике боя и состояниях игры.

---

# Библиотека

Использовать библиотеку fast-check.

---

# Новый каталог

Создать каталог:

tests/property

---

# Property tests

Добавить тесты для следующих свойств:

1. HP существ никогда не может быть меньше 0.

2. Существа с HP = 0 должны удаляться с поля.

3. Существо не может атаковать более одного раза за ход.

4. Размер колоды не может быть отрицательным.

5. Mana игрока не может быть отрицательной.

---

# Генерация данных

Использовать fast-check для генерации:

* случайных существ
* случайных HP
* случайных keywords

---

# Симуляция

Использовать существующие функции движка:

makeCreature
makeState
attackCreature
attackPlayer

---

# Проверка

Запускать property тесты минимум 500–1000 раз.

---

# Ограничения

Не изменять существующие unit tests и regression tests.

Property tests должны быть дополнительным уровнем проверки.
