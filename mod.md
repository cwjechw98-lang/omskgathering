Перед началом изменений необходимо стабилизировать архитектуру интерфейса игры.

Проект уже содержит:

* игровой движок
* систему карт
* систему визуальных эффектов
* систему логирования разработки
* git workflow

Все изменения должны использовать существующие правила проекта:

* существующая система логов
* существующий процесс commit
* существующий процесс push

Не изменять формат логирования.

---

# Цель

Стабилизировать интерфейс карточной игры и подготовить его к дальнейшему развитию.

Основные проблемы текущего интерфейса:

* элементы перекрывают друг друга
* карты ломают layout при hover
* эффекты не имеют строгих слоев
* рука игрока плохо читается
* отсутствует визуальная структура слотов существ

---

# Важно

Работу выполнять по этапам.

После каждого этапа:

* обновить лог разработки
* сделать commit
* сделать push

4. какие компоненты отвечают за:

* effects

Определить:

* какие компоненты безопасно менять
* какие связаны с игровым движком

Не изменять код на этом этапе.

---

# ЭТАП 2 — Архитектура карты

Исправить структуру компонента карты.

Каждая карта должна иметь структуру:

CardSlot
CardContainer
CardVisual

CardSlot
— определяет позицию карты на поле
— фиксированный размер

CardContainer
— отвечает за hover
— отвечает за drag
— отвечает за scale

CardVisual
— artwork
— текст
— рамка карты
— foil эффект
— glow эффект
— статусные иконки

CardVisual должен иметь:

overflow: hidden

Hover масштабирование должно происходить только у CardContainer.

---

# ЭТАП 3 — Система слоёв интерфейса

Создать централизованную систему z-index.

Слои:

background
board
cardSlots
cards
cardEffects
combatEffects
ui
hover
overlay

Использовать эти слои во всех компонентах.

Проверить:

* эффекты не перекрывают UI
* overlay всегда выше остальных элементов
* hover карты не ломают layout

---

# ЭТАП 4 — Board Slots

Реализовать систему слотов существ.

Каждая сторона поля должна иметь:

7 слотов.

Layout:

[slot][slot][slot][slot][slot][slot][slot]

Использовать CSS grid.

grid-template-columns: repeat(7, 1fr)

Слот должен:

* центрировать карту
* иметь placeholder если пустой

---

# ЭТАП 5 — Attack Lanes

Каждый слот формирует вертикальную дорожку атаки.

При атаке:

* подсветить lane
* подсветить цель

Это только визуализация.

Игровую механику не изменять.

---

# ЭТАП 6 — Arc Hand Layout

Реализовать дуговое расположение карт руки.

Карты должны располагаться по дуге.

Использовать:

rotate(angle)
translateY(offset)

Центр дуги должен быть в центре HandZone.

При hover:

* карта увеличивается
* rotation убирается
* карта поднимается вверх

При drag карта должна временно выходить из дуги.

---

# ЭТАП 7 — Overlay для выбора карт

Все эффекты типа:

* choose card
* look top cards
* discover

должны использовать modal overlay.

Overlay должен:

* затемнять фон
* блокировать взаимодействие с полем
